Purpose

This project is meant to showcase smaller frontend projects like a portfolio. It's a project I can work on in my free time to inventory my current skills, starting with HTML, CSS, and JavaScript, and later transitioning to frameworks.

Goals

  • Create an SPA hosted on GitHub pages which dynamically loads components and modules.
  • Have a responsive design that works down to a screen width of 320 pixels.
  • Build a custom navbar from scratch responsible for navigation between modules.
  • Solidify my current skills.
  • Learn new skills.
  • Have fun!

Current Features

  • Navbar

    The navbar has two positions based on screen width. Regardless of position it has a carousel to navigate between modules via their icons - placeholders for now -, and an expandable area which displays module names - placeholders for now

    All assets, including the navbar design and interactions, are being created from scratch by my, without the use of frameworks like Bootstrap, relying instead on my own CSS and JavaScript skills

Future Features

  • Minigames

    I intend to make a series of retro type minigames as their own modules. For example I have the intention of creating a button clicker, tic tac toe, snake, and pong.

  • Integration with APIs

    I have a collection of APIs which I intend to use to populate data in certain modules - my first goal is to make a team builder using PokeAPI
